The application uses TLS for all client-server communication, configured via a PKCS12 keystore loaded by Spring Boot and a corresponding TLS-enabled Vite dev server on the client. For development, a self-signed RSA 2048-bit certificate is used, which the browser flags as untrusted on first visit — this is expected behavior, since browsers only auto-trust certificates issued by recognized Certificate Authorities. In a production deployment, the keystore would contain a CA-issued certificate (e.g. via Let's Encrypt) bound to the application's real domain name, eliminating the warning. The TLS configuration itself (cipher suites, protocol versions, key exchange) is identical in both cases — only the certificate's chain of trust differs.
